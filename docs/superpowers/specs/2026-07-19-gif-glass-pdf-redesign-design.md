# GIF 배경·글래스 UI·PDF 진단 개편 설계

## 목표

첨부된 저폴리 숲 GIF를 로그인 화면의 중심 배경으로 사용하고, 학생·교사 전체 화면에서는 기존 배경이 더 선명하게 보이도록 표면 투명도와 오버레이를 조정한다. 질문 작성 흐름은 질문을 먼저 적은 뒤 유형을 선택하도록 바꾸며, 교사의 AI 질문 진단 결과는 HWPX 대신 웹앱의 시각 언어를 반영한 PDF로 내려받게 한다.

## 확정된 시각 방향

- **배경:** 로그인은 `image-6.gif`를 `/backgrounds/login-forest.gif`로 복사해 전체 화면 `cover` 배경으로 쓴다. 학생·교사 화면은 기존 저폴리 포스터를 유지한다.
- **글래스 표면:** 로그인 카드는 `rgba(255,255,255,.68)`과 강한 blur, 얇은 흰 테두리, 부드러운 그림자를 사용한다. 로그인 후 화면은 큰 흰색 덮개 대신 `rgba(255,255,255,.74)` 수준의 유리 표면만 필요한 정보 단위에 적용한다.
- **배경 노출:** 학생·교사 배경 오버레이 불투명도를 낮추고, 모바일에서도 표면을 불투명 흰색으로 되돌리지 않는다. 텍스트 가독성은 표면 내부의 어두운 글자와 국소 그림자로 확보한다.
- **상단 색 띠 제거:** `StorySurface`의 student/teacher/warm/neutral `border-top`을 제거한다. 버튼·카드는 상단에 별도 주황/보라/파랑 선이 생기지 않는다.
- **색과 타이포:** 기존 Pretendard, 학생 보라, 교사 파랑을 유지하되 배경의 초록·햇빛이 보조색으로 드러나게 한다. 색은 구조와 상태 표시에만 사용한다.
- **대표 요소:** 움직이는 숲 배경과 반투명 로그인 카드의 대비를 한 가지 핵심 시각 요소로 삼는다.

## 화면 및 상호작용

### 로그인

- GIF 배경은 반복 재생되며 화면을 채운다.
- `prefers-reduced-motion` 사용자는 GIF 대신 정적 기존 포스터를 본다.
- 로그인 카드, 입력창, 오류 메시지는 유리 표면 위에서 충분한 대비를 가진다.

### 학생·교사 공통

- `StorySurface`는 상단 색 띠 없이 반투명 유리 표면이 된다.
- 학생·교사 배경 위 전체 색 오버레이를 약하게 해 배경 지형이 보이게 한다.
- 카드가 중첩될 때 내부 카드는 더 가볍고 투명한 표면을 사용한다.

### 학생 질문 작성

1. 책 정보를 확인한다.
2. 첫 번째 단계에서 질문을 작성한다.
3. 두 번째 단계에서 사실/궁금/라면 질문 유형을 선택한다.
4. 질문과 유형이 모두 있을 때 제출 버튼이 활성화된다.

라디오 그룹의 방향키·Home·End 키보드 조작은 유지한다.

### 학생 상단 내비게이션

`책 목록`과 `내 기록`을 같은 `navItem` 컴포넌트와 동일한 유리 버튼 스타일로 표시한다. 각 항목은 현재 경로에서 동일한 활성 상태를 가진다.

## PDF 진단

- API 경로는 `/api/download-pdf`로 바꾸고 HWPX API는 제거한다.
- 입력은 학생 이름, 총평, 강점, 성장 방향, 질문 수 및 유형별 통계다.
- 서버에서 `pdf-lib`와 `@pdf-lib/fontkit`으로 A4 PDF를 생성한다.
- 한글 지원을 위해 저장소에 포함한 Pretendard 글꼴을 subset 임베딩한다.
- 표지 헤더, 학생명, 생성일, 세 가지 진단 섹션, 질문 유형 통계, 푸터/페이지 번호를 포함한다.
- 긴 문장은 글꼴 폭을 기준으로 줄바꿈하고 페이지가 부족하면 새 페이지를 만든다.
- 응답은 `application/pdf`와 UTF-8 파일명의 `Content-Disposition: attachment`를 사용한다.

## 오류 처리

- 필수 진단 값이 없으면 PDF API는 400 JSON 오류를 반환한다.
- 생성 실패는 서버 로그에 기록하고 500 JSON 오류를 반환한다.
- 클라이언트는 `fetch`로 PDF를 받은 뒤 Blob URL로 다운로드하며, 실패 시 사용자에게 오류 알림을 표시한다.

## 검증

- UI 계약 테스트로 GIF 경로, 글래스 표면, 상단 띠 제거, 질문 단계 순서, 내 기록 버튼 동등성을 검증한다.
- PDF 단위 테스트로 `%PDF` 시그니처, 한글 텍스트 입력, 파일명 정리, 통계 포함을 검증한다.
- 전체 `npm test`, `npm run lint`, `npm run build`, 자산 검증을 실행한다.
- 샘플 PDF를 생성하고 PNG로 렌더링해 한글 깨짐, 잘림, 정렬, 페이지 전환을 시각 검수한다.
- 로컬 브라우저에서 로그인·학생·교사 주요 화면을 데스크톱과 모바일 너비로 확인한다.

## 범위 밖

- Supabase 스키마와 질문 유형 enum은 변경하지 않는다.
- Gemini 모델과 분석 프롬프트는 변경하지 않는다.
- 새로운 AI 생성 이미지나 기존 브랜드 로고는 만들지 않는다.
